# adaptive-product-designer

> Универсальный репозиторий дизайн-скиллов для работы с LLM.
> Покрывает 8 доменов: продуктовый дизайн, дизайн-системы, UX writing, UX-исследования, продуктовая аналитика, UX/UI теория, графический дизайн, брендинг.

---

## Что это

Репозиторий, превращающий любую LLM (в первую очередь Claude) в дизайн-ассистента, который:

- помнит твой профиль, стиль и принципы (`/personal/`);
- знает структурированные методы по 8 доменам (`/skills/`);
- умеет применять зеркала проверенных скиллов Anthropic (`/mirrors/`);
- следует логике роутинга — подгружает только то, что нужно под запрос (`ROUTER.md`);
- выдаёт результат в формате готовых deliverables (`/templates/`).

LLM не пересказывает скиллы — она их **применяет**.

---

## Как использовать за 3 шага

### 1. Заполни `/personal/`

Без этого репозиторий работает универсально, а не «под тебя».

- `personal/profile.md` — кто ты, индустрия, инструменты.
- `personal/tone-of-voice.md` — твой стиль, примеры текстов.
- `personal/design-principles.md` — твои принципы.
- `personal/current-projects.md` — над чем работаешь.

### 2. Вставь Project Instructions в Claude Project

Скопируй содержимое `PROJECT_INSTRUCTIONS.md` в настройки своего Claude Project (`Settings → Custom instructions`). Это активирует роутер.

### 3. Пиши обычные запросы

Дальше ничего настраивать не нужно. Пиши как обычно:

- «Сделай ревью этого экрана»
- «Помоги с микрокопи для empty state»
- «Какие метрики поставить на новую фичу»
- «Спецификация компонента Button для разработки»

LLM сама определит домен, подгрузит нужные скиллы (через `web_fetch` raw-ссылок из этого репозитория) и применит их.

## Структура

```
adaptive-product-designer/
├── README.md                 ← ты здесь
├── INDEX.md                  ← каталог всех ресурсов (локальных и внешних)
├── ROUTER.md                 ← логика подключения скиллов
├── SCHEMA.md                 ← формат SKILL.md
├── PROJECT_INSTRUCTIONS.md   ← шаблон для Claude Project
├── CONTRIBUTING.md           ← как добавлять/обновлять
├── CHANGELOG.md
│
├── personal/                 ← твой персональный контекст
├── skills/                   ← скиллы по 8 доменам
├── mirrors/                  ← зеркала проверенных внешних скиллов
├── templates/                ← шаблоны deliverables
├── frameworks/               ← быстрые справочники (cheatsheets)
└── evals/                    ← тестовые запросы для проверки роутинга
```

Подробно — в `INDEX.md`.

## Принципы репозитория

1. **Локальное побеждает внешнее.** Свои скиллы перебивают зеркала, зеркала — внешние ссылки.
2. **Персональное побеждает любое общее.** `/personal/` всегда применяется первым.
3. **Прогрессивная загрузка.** В контекст идёт только то, что триггернулось запросом.
4. **Прозрачность.** В конце каждого ответа LLM перечисляет использованные скиллы.
5. **Английский для матчинга, русский для контента.** Frontmatter — EN, тело — RU, термины индустрии — EN.

## Статус

Репозиторий развивается по фазам:

- **Фаза 0 — Фундамент** (завершена): `SCHEMA`, `ROUTER`, `/personal/`, README, INSTRUCTIONS, зеркала Anthropic, базовые шаблоны.
- **Фаза 1 — Закрытие дыр:** product-analytics, graphic-design, branding (домены без покрытия).
- **Фаза 2 — Систематизация теории:** ux-ui-theory.
- **Фаза 3 — Углубление практик:** ux-research, ux-writing с твоей адаптацией.
- **Фаза 4 — Доводка:** product-design, design-systems с твоими workflows, evals, v1.0.

Текущий статус каждого скилла — в его frontmatter (`status: stable | draft | experimental`).

---

## Как развивать

Любые изменения — через `CONTRIBUTING.md`. Главные правила:

- Новый скилл = новая директория с `SKILL.md` по `SCHEMA.md`.
- Обновление зеркала Anthropic — не редактируй оригинал, дополнения в `EXTENSIONS.md`.
- Каждое изменение — запись в `CHANGELOG.md`.

---

## Атрибуция

- Зеркала из `/mirrors/anthropic/` — это снапшоты [anthropic/knowledge-work-plugins](https://github.com/anthropics/knowledge-work-plugins) с указанием лицензии и даты.
- Все ссылки в `INDEX.md` ведут на оригиналы.
- Собственные скиллы в `/skills/` — авторские, лицензия в `LICENSE`.
