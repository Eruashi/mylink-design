# mylink-design

> Универсальный репозиторий дизайн-скиллов для работы с LLM, рассчитанный на использование **командой**.
> Покрывает 8 доменов: продуктовый дизайн, дизайн-системы, UX writing, UX-исследования, продуктовая аналитика, UX/UI теория, графический дизайн, брендинг.

---

## Что это

Репозиторий, превращающий любую LLM (в первую очередь Claude) в дизайн-ассистента команды, который:

- помнит **командный** контекст: компания, продукты, общие принципы, бренд-голос (`/team/`);
- помнит **личный** контекст каждого дизайнера: специализация, стиль письма, личные принципы (`/people/<имя>/`);
- знает структурированные методы по 8 доменам (`/skills/`);
- применяет зеркала проверенных скиллов Anthropic (`/mirrors/`);
- следует логике роутинга — подгружает только то, что нужно под запрос (`ROUTER.md`);
- выдаёт результат в формате готовых deliverables (`/templates/`).

LLM не пересказывает скиллы — она их **применяет**, учитывая и команду, и того конкретного человека, который сейчас задаёт вопрос.

## Как использовать команде

### Один раз — настройка команды

Заполните `/team/` — командный контекст:

- `team/profile.md` — кто мы как команда
- `team/design-principles.md` — командные принципы и обязательные стандарты
- `team/current-projects.md` — каталог активных проектов
- `team/brand-tone.md` — бренд-голос продукта (опционально)

### Каждый член команды — onboarding

Каждый дизайнер заводит свою личную папку и подключает её к своему Claude Project. Подробная инструкция — в [`people/README.md`](people/README.md). Кратко:

1. Создай `/people/<своё-имя>/` и скопируй туда файлы из `/people/_template/`.
2. Заполни их — особенно `tone-of-voice.md` с примерами своих текстов.
3. Открой свой Claude Project → Settings → Custom instructions → вставь содержимое `PROJECT_INSTRUCTIONS.md`.
4. Замени `<REPLACE_WITH_YOUR_USERNAME>` на имя своей папки.

После этого пиши обычные запросы. LLM сама определит, что грузить.

```

├── CONTRIBUTING.md           ← как добавлять/обновлять
├── CHANGELOG.md
│
├── team/                     ← командный контекст (общий для всех)
│   ├── profile.md
│   ├── design-principles.md
│   ├── current-projects.md
│   └── brand-tone.md
│
├── people/                   ← личный контекст каждого дизайнера

├── skills/                   ← скиллы по 8 доменам (в работе)

├── templates/                ← шаблоны deliverables
├── frameworks/               ← быстрые справочники
└── evals/                    ← тестовые запросы для проверки роутинга
```

## Принципы репозитория

1. **Иерархия:** `/people/<active-user>/` → `/team/` → `/skills/meta/` → `/skills/<domain>/` → `/mirrors/` → `INDEX.md`.
2. **Локальное побеждает внешнее.** Свои скиллы перебивают зеркала, зеркала — внешние ссылки.
3. **Личное побеждает командное на уровне стиля.** Командное побеждает на уровне обязательных стандартов.
4. **Прогрессивная загрузка.** В контекст идёт только то, что триггернулось запросом.
5. **Прозрачность.** В конце каждого ответа LLM перечисляет использованные скиллы.
6. **Английский для матчинга, русский для контента.** Frontmatter — EN, тело — RU, термины индустрии — EN.

## Как развивать

Любые изменения — через `CONTRIBUTING.md`. Главные правила:

- Новый скилл = новая директория с `SKILL.md` по `SCHEMA.md`.
- Обновление зеркала Anthropic — не редактируй оригинал, дополнения в `EXTENSIONS.md`.
- Каждое изменение — запись в `CHANGELOG.md`. ⏸ _(ведение на паузе до Phase 4 — см. блок статуса в `CHANGELOG.md`)_
- Изменения в `/team/` влияют на всю команду — обсуждайте, прежде чем коммитить.
- Изменения в `/people/<имя>/` — личная зона, владелец сам решает.

## Атрибуция

- Все ссылки в `INDEX.md` ведут на оригиналы.
- Собственные скиллы в `/skills/` — авторские, лицензия в `LICENSE`.
